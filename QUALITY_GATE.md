# QUALITY_GATE.md

## Purpose
記事を完成版として出力する前に、品質・形式・リンク・アフィリエイト分岐を最終確認する。

このファイルは、記事完成前の最終チェック専用である。

AGENTS.mdやOUTPUT_FORMAT.mdの内容を実行した後、最後に必ずQUALITY_GATE.mdを通過させる。

---

# =========================
# Final Gate Principle
# =========================

## 最重要原則

記事内容が良くても、以下のどれかを満たしていない場合は完成版として出力しない。

・Markdown形式が崩れている  
・コピー用BOXに入っていない  
・本文内リンクが実際に挿入されていない  
・アフィリエイトあり/なしの分岐が間違っている  
・H3本文が薄い  
・メリット・デメリットが箇条書きだけ  
・末尾項目が不足している  
・自己評価だけが高く、実態が伴っていない  

---

# =========================
# Gate 1: Output Format Check
# =========================

## チェック項目

□ 完成記事は ```markdown のコピー用BOX内に入っているか  
□ 記事タイトルは # で始まっているか  
□ 導入文は ## 見出しになっているか  
□ 結論は ## 見出しになっているか  
□ 要約は ## 見出しになっているか  
□ 本文は ## 見出しになっているか  
□ まとめは ## 見出しになっているか  
□ Q&Aは ## 見出しになっているか  
□ H3は ### で統一されているか  
□ プレーンテキスト見出しが残っていないか  

## 完成不可条件

以下がある場合は完成版として出力しない。

```text
記事1（完成版）
導入文
結論
要約
本文
まとめ
Q&A
```

必ず以下に直す。

```markdown
# 【記事1】タイトル

## 導入文

## 結論

## 要約

## 本文

## まとめ

## Q&A
```

# =========================
# Gate 2: Internal Link Check
# =========================

## チェック項目

□ 5記事作成時に5記事内部リンク設計があるか  
□ 5記事分のパーマリンクがあるか  
□ 5記事分のURLがあるか  
□ 本文内に内部リンクが2〜3本入っているか  
□ 内部リンクはURL付きMarkdown形式か  
□ アンカーテキストが自然か  
□ 「こちら」だけになっていないか  
□ 末尾の内部リンク案だけで終わっていないか  

## 正しい形式

```markdown
noteの書き方を詳しく知りたい方は、次の記事も参考になります。

[noteで読まれる記事の書き方を読む](https://jiyoujin.com/note-kakikata-template/)
```

## 完成不可条件

以下の場合は完成版として出力しない。

・内部リンク案だけがあり、本文中にリンクがない  
・記事タイトルだけでURLがない  
・URLだけでアンカーテキストがない  
・「こちら」だけのリンクになっている  
・5記事クラスターなのに内部リンク設計がない

# =========================
# Gate 3: Affiliate Branch Check
# =========================

## アフィリエイトあり記事

以下を満たす必要がある。

□ 記事冒頭に広告・PR表記があるか  
□ 本文内にアフィリエイトリンクが実際に入っているか  
□ アフィリエイトリンクはMarkdown形式か  
□ アフィリエイトリンクがCTA案だけで終わっていないか  
□ 使用したアフィリエイトサービスが明記されているか  
□ 使用したアフィリエイトリンクが明記されているか  
□ 本文内アフィリエイトリンク挿入数が実際の数と一致しているか  

## 正しい形式

```markdown
※本記事には広告・PRが含まれます。

# 【記事1】タイトル

本文...

[サービス名の詳細を確認する](AFFILIATE_LINK_サービス名)
```

## 完成不可条件

以下の場合は完成版として出力しない。

・PR表記がない  
・本文内にアフィリエイトリンクがない  
・末尾にリンク挿入数だけ書いている  
・CTA案にだけリンクがある  
・本文内リンクが `(AFFILIATE_LINK_サービス名)` の文字列だけで、Markdownリンクになっていない  

---

## アフィリエイトなし記事

以下を満たす必要がある。

□ PR表記が入っていないか  
□ AFFILIATE_LINK_〇〇 が入っていないか  
□ 使用したアフィリエイトサービスが「なし」になっているか  
□ 使用したアフィリエイトリンクが「なし」になっているか  
□ 本文内アフィリエイトリンク挿入数が「0本」になっているか  
□ アフィリエイト導線が「なし」になっているか  

## 完成不可条件

以下の場合は完成版として出力しない。

・アフィリエイトなし記事なのにPR表記がある  
・アフィリエイトなし記事なのに仮リンクがある  
・アフィリエイトなし記事なのに購入CTAがある  
・使用したアフィリエイトサービス欄が曖昧

# =========================
# Gate 4: H3 Depth Check
# =========================

## チェック項目

□ H3直後が一言で終わっていないか  
□ H3本文が100文字未満になっていないか  
□ H3本文が箇条書きだけになっていないか  
□ H3のうち70%以上に、結論・理由・具体例・注意点のいずれかが含まれているか  
□ 薄いH3を最低3か所以上補強したか  

## 完成不可条件

以下のようなH3は未完成。

```markdown
### 1. 発信テーマ

誰に向けて書くかを決めます。
```

以下のように補強する。

```markdown
### 1. 発信テーマを決める

noteを始める前に、まず「誰に向けて何を書くのか」を決めておくことが大切です。発信テーマが曖昧なままだと、記事ごとに内容がぶれやすく、読者がフォローする理由も弱くなります。たとえば、副業初心者向けに書くなら、体験談だけでなく「何に悩み、どう解決したか」まで書くと読者に役立つ記事になります。
```

# =========================
# Gate 5: Benefit & Drawback Check
# =========================

## チェック項目

□ メリット・デメリットが独立見出しであるか  
□ 「### メリット」「### デメリット」だけで終わっていないか  
□ 各項目が「### メリット1：〜」形式になっているか  
□ 各項目が250〜400文字以上で説明されているか  
□ メリットだけでなくデメリットも正直に書いているか  
□ 初心者がどう判断すべきかまで書いているか  

## 完成不可条件

以下は未完成。

```markdown
## メリット・デメリット

### メリット

- 始めやすい
- 無料で使える
- 投稿しやすい

### デメリット

- 読まれにくい
- 継続が必要
```

以下のように修正する。

```markdown
## メリット・デメリット

### メリット1：初期コストが低く、すぐ始められる

noteは無料で始められるため、文章発信のハードルが低いのが大きなメリットです。WordPressのようにサーバー契約や初期設定をする必要がなく、思い立った日に投稿まで進められます。ただし、始めやすい分ライバルも多いため、テーマ設計や継続投稿で差をつける必要があります。

### デメリット1：書くだけでは読まれにくい

noteは投稿しやすい一方で、公開しただけで多くの読者に届くわけではありません。タイトル、導入文、投稿頻度、SNSでの共有など、読まれるための工夫が必要です。初心者は「良い文章を書けば自然に伸びる」と考えがちですが、実際には改善を前提に運用することが大切です。
```

# Gate 6: Required Final Items Check
# =========================

## チェック項目

記事末尾に以下がすべてあるか確認する。

□ メタディスクリプション  
□ タグ  
□ パーマリンク  
□ URL  
□ 画像用短縮タイトル  
□ タイトル入りアイキャッチ画像プロンプト  
□ 記事内画像プロンプト  
□ 5記事内部リンク設計  
□ 本文内に挿入した内部リンク  
□ 使用したアフィリエイトサービス  
□ 使用したアフィリエイトリンク  
□ 本文内アフィリエイトリンク挿入数  
□ リンク挿入位置  
□ アフィリエイト導線  
□ CTA案  
□ 記事スコア  

不足している項目がある場合は完成版として出力しない。

# =========================
# Gate 7: Score Check
# =========================

## ルール

記事スコアは、自己評価ではなく、以下の条件を満たしてから付ける。

## 95点以上にしてよい条件

□ Markdown BOXに入っている  
□ 見出し形式が正しい  
□ 本文内内部リンクがある  
□ アフィリエイト分岐が正しい  
□ H3本文が薄くない  
□ メリット・デメリットが深掘りされている  
□ 末尾項目が揃っている  
□ 画像プロンプトが具体的  
□ Q&Aが5個以上ある  
□ 読者の次の行動が明確  

## 95点以上にしてはいけない条件

・Markdown BOXに入っていない  
・内部リンクが本文中にない  
・アフィリエイトリンクが本文中にない  
・H3が短い  
・メリット・デメリットが箇条書きだけ  
・末尾項目が不足  
・記事スコアだけ高い  
・記事全体がテンプレ的で薄い

# =========================
# Final Decision
# =========================

## 完成として出力してよい場合

以下をすべて満たしている場合のみ完成版として出力する。

1. OUTPUT_FORMAT.mdに従っている
2. Markdown BOXに入っている
3. 見出し形式が正しい
4. 内部リンクが本文中に実挿入されている
5. アフィリエイトあり/なしの分岐が正しい
6. H3本文が十分に深い
7. メリット・デメリットが深掘りされている
8. 末尾項目がすべてある
9. 記事スコアが妥当
10. WordPressへ貼り付けられる状態である

## 完成として出力してはいけない場合

以下のいずれかに当てはまる場合は、完成版として出力しない。

・形式が崩れている  
・本文が薄い  
・リンクが本文中にない  
・アフィリエイト分岐が間違っている  
・末尾項目が不足している  
・95点評価が甘い  
・コピーしづらい
