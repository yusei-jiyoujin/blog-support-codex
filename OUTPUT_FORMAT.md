# OUTPUT_FORMAT.md

## Purpose
記事の最終出力形式を統一し、ユーザーがWordPressへそのまま貼り付けられる状態にする。

このファイルは、記事本文の品質ではなく、**出力形式・コピーしやすさ・Markdown構造・リンク形式・末尾項目**を管理する。

AGENTS.mdよりも、最終出力形式に関してはこのOUTPUT_FORMAT.mdを優先する。

---

# =========================
# Output Priority Rule
# =========================

## 最重要ルール

完成記事は、必ず以下の条件を満たす。

1. 完成記事は必ず ```markdown のコピー用BOX内に出力する
2. 記事タイトルは必ず # で始める
3. 導入文・結論・要約・本文・まとめ・Q&A は必ず ## 見出しにする
4. H3見出しは必ず ### にする
5. 5記事内部リンクは、本文中にURL付きMarkdownリンクとして実際に挿入する
6. アフィリエイトリンクがある場合のみ、広告・PR表記を記事冒頭に入れる
7. アフィリエイトリンクがある場合のみ、本文中にURL付きMarkdownリンクとして実際に挿入する
8. アフィリエイトリンクがない記事では、広告・PR表記や仮リンクを入れない
9. 記事末尾に指定されたメタ情報をすべて出力する
10. 記事スコアを必ず出力する

# =========================
# Markdown Heading Rule
# =========================

## 必須見出し構造

記事は必ず以下の見出し構造で出力する。

```markdown
# 【記事1】記事タイトル

## 導入文

## 結論

## 要約

## 本文

## まとめ

## Q&A
```

本文内では、必要に応じて以下を使用する。

```markdown
## 大見出し

### 小見出し

#### 補足見出し
```

---

## 禁止形式

以下のようなプレーンテキスト見出しは禁止。

```text
導入文
結論
要約
本文
まとめ
Q&A
```

以下も禁止。

```text
記事1（完成版）
noteの始め方完全ガイド
noteを始める前に決めるべき3つ
1. 発信テーマ
```

必ず以下のように修正する。

```markdown
# 【記事1】noteの始め方完全ガイド

## 導入文

## noteを始める前に決めるべき3つ

### 1. 発信テーマ
```

# =========================
# Internal Link Output Rule
# =========================

## 目的
5記事クラスター内の内部リンクを、本文中に実際に挿入する。

## ルール

5記事内部リンク設計を作成した場合、内部リンクは末尾に案として出すだけでは不十分。

必ず本文中に、URL付きMarkdownリンクとして挿入する。

## 正しい形式

```markdown
noteで読まれる記事の型を知りたい方は、次にこちらの記事も参考になります。

[noteで読まれる記事の書き方を読む](https://jiyoujin.com/note-kakikata-template/)
```

## 禁止形式

```text
書き方を深掘りしたい方は「noteで読まれる記事の書き方」
```

```text
noteで読まれる記事の書き方
```

```text
こちら
```

```text
関連記事：noteで読まれる記事の書き方
```

# =========================
# Affiliate Link Output Rule
# =========================

## 適用条件

以下の場合のみ、アフィリエイトリンクを本文中に挿入する。

・ユーザーがアフィリエイトリンクを入力した場合  
・アフィリエイトサービスが入力され、仮リンクを使用する場合  

アフィリエイトリンクがない記事では、このルールは適用しない。

---

## 広告・PR表記

アフィリエイトリンクがある場合は、記事冒頭に必ず以下のいずれかを入れる。

```markdown
※本記事には広告・PRが含まれます。
```

または

```markdown
※この記事はアフィリエイト広告を利用しています。
```

広告・PR表記は、タイトル直前またはタイトル直後に入れる。

---

## 正しい本文内リンク形式

```markdown
導入前にサービス内容や料金体系を確認したい方は、公式情報を確認してから判断すると安心です。

[サービス名の詳細を確認する](AFFILIATE_LINK_サービス名)
```

または

```markdown
導入前にサービス内容や料金体系を確認したい方は、公式情報を確認してから判断すると安心です。

[サービス名の詳細を確認する](https://example.com/affiliate-link)
```

---

## 禁止形式

```text
サービス名の詳細を確認する（AFFILIATE_LINK_サービス名）
```

```text
本文内アフィリエイトリンク挿入数：2本
```

だけを末尾に書き、本文中にリンクがない状態は禁止。

## アフィリエイトなし記事のルール

アフィリエイトサービスやアフィリエイトリンクがない記事では、以下のように出力する。

```markdown
0本

なし

アフィリエイト導線：
なし
```

## 禁止事項

アフィリエイトなし記事では以下を禁止する。

・広告・PR表記を入れる  
・AFFILIATE_LINK_〇〇 を入れる  
・存在しないアフィリエイトURLを推測する  
・無理に購入導線を作る  
・CTAを商品購入寄りにする

# =========================
# Completion Rule
# =========================

以下のいずれかに該当する場合は、完成版として出力しない。

・コピー用Markdown BOXに入っていない  
・記事タイトルが # になっていない  
・導入文、結論、要約、本文、まとめ、Q&A が ## になっていない  
・本文内内部リンクがURL付きMarkdown形式になっていない  
・アフィリエイトリンクがあるのにPR表記がない  
・アフィリエイトリンクがあるのに本文内リンクがない  
・アフィリエイトなし記事なのにPR表記や仮リンクが入っている  
・末尾の必須項目が不足している  
・記事スコアが95点以上でも形式ルールを満たしていない
