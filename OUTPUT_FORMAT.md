# OUTPUT_FORMAT.md

## Purpose

記事の最終出力形式を統一し、WordPressへ貼り付けた際に、本文が表・不要なブロック・管理情報として崩れない状態にする。

このファイルは、記事本文の品質ではなく、**出力形式・コピーしやすさ・WordPress貼り付け範囲・リンク形式・管理情報の分離**を管理する。

AGENTS.mdよりも、最終出力形式に関してはこの OUTPUT_FORMAT.md を優先する。

---

# =========================
# Output Priority Rule
# =========================

## 最重要ルール

完成記事は、必ず以下の2つに分けて出力する。

1. WordPress本文貼り付け用
2. 管理情報用

## 絶対ルール

・WordPress本文貼り付け用には、読者に見せる記事本文だけを入れる  
・WordPress本文貼り付け用は、Markdown記法を含むプレーンテキストとして出力する  
・WordPress本文貼り付け用は、.mdではなく .txt として扱う  
・管理情報用には、メタ情報・画像プロンプト・内部リンク設計・記事スコアなどを入れる  
・WordPress本文貼り付け用と管理情報用を混ぜない  
・5記事テーマ案、パーマリンク案、URL案、記事スコア、画像プロンプトをWordPress本文内に入れない  
・WordPress本文貼り付け用には、表形式・Markdown表・タブ区切り・横並び比較表を使わない  
・表に見える情報は、H2/H3・箇条書き・段落で表現する  
・WordPressに貼るのは「WordPress本文貼り付け用」のみとする  

---

# =========================
# WordPress Plain Text Output Rule
# =========================

## 目的

WordPress貼り付け時に、記事本文が表ブロックとして挿入されることを防ぐ。

## 基本ルール

WordPress本文貼り付け用は、Markdown記法を含むプレーンテキストとして出力する。

## 出力ファイル想定

・WordPress本文用：article-body.txt  
・管理情報用：article-info.txt  

## 注意

WordPress本文用は .md ファイルとして出力しない。  
Markdown記法は使ってよいが、ファイル形式・出力名・貼り付け対象は .txt として扱う。

# =========================
# WordPress Body Separation Rule
# =========================

## 目的

WordPressに貼り付けた際、記事本文が表や不要なブロックとして挿入されるのを防ぐ。

## 出力構成

完成記事は、以下の2ブロックに分ける。

```text
【1】WordPress本文貼り付け用（.txt）

【2】管理情報用
```

---

・アフィリエイト導線

# =========================
# WordPress Body Output Rule
# =========================

## 目的

WordPress本文欄に貼るための記事本文だけを、余計な管理情報なしで出力する。

## 基本形式

WordPress本文貼り付け用は、以下の見出し構造を基本にする。

```text
# 【記事1】記事タイトル

## 導入文

本文

## 結論

本文

## 要約

本文

## 本文

## 大見出し

本文

### 小見出し

本文

## まとめ

本文

## Q&A

### Q1. 質問

回答
```

---

## 見出しルール

・記事タイトルは `#` を使う  
・導入文、結論、要約、本文、まとめ、Q&A は `##` を使う  
・本文内の大見出しも `##` を使う  
・小見出しは `###` を使う  
・補足見出しが必要な場合のみ `####` を使う  
・プレーンテキストだけの見出しは禁止  

---

## 禁止形式

以下のような形式は禁止。

```text
記事1（完成版）
導入文
結論
要約
本文
まとめ
Q&A
```

以下も禁止。

```text
noteを始める前に決めるべき3つ
1. 発信テーマ
2. 投稿頻度
3. ゴール
```

正しくは以下のようにする。

```text
## noteを始める前に決めるべき3つ

### 1. 発信テーマを決める

本文

### 2. 投稿頻度を決める

本文

### 3. ゴールを決める

本文
```

# =========================
# Copy Rule
# =========================

## 目的

WordPress貼り付け時に、表ブロック化・コードブロック化・不要ブロック化を防ぐ。

## ルール

・WordPress本文貼り付け用は、コードブロックではなく通常テキストとして出力する  
・WordPress本文貼り付け用を ```markdown で囲まない  
・WordPress本文貼り付け用を ```text で囲まない  
・WordPress本文貼り付け用を .md ファイルとして扱わない  
・WordPress本文貼り付け用は .txt のプレーンテキストとしてコピーする  
・管理情報用はコードブロックやMarkdown形式でもよい  
・WordPress本文に貼るのは、必ず「WordPress本文貼り付け用（.txt）」だけにする  

## 禁止事項

・WordPress本文貼り付け用をコードブロックで囲む  
・WordPress本文貼り付け用と管理情報用を1つのコピー範囲にまとめる  
・WordPress本文貼り付け用に5記事テーマ案やURL一覧を混ぜる  
・WordPress本文貼り付け用に記事スコアや画像プロンプトを混ぜる  

---

# =========================
# Table Avoidance Rule
# =========================

## 目的

WordPressに貼り付けた際、記事本文が表として挿入されるのを防ぐ。

## 禁止事項

WordPress本文貼り付け用では、以下を使わない。

・Markdown表  
・HTMLテーブル  
・タブ区切り  
・縦線 `|` を使った比較表  
・項目を横並びにした表現  
・スプレッドシート風の整列  
・複数列の比較表  
・5記事テーマ案やURL案の一覧を本文冒頭に入れること  

## 禁止例

```markdown
| 項目 | 内容 |
|---|---|
| メリット | 始めやすい |
| デメリット | 継続が必要 |
```

## 正しい代替形式

表ではなく、H3と箇条書きで表現する。

```text
### メリット1：始めやすい

本文

### デメリット1：継続には工夫が必要

本文
```

比較が必要な場合も、以下のように文章と箇条書きで表現する。

```text
## noteとWordPressの違い

### noteが向いているケース

本文

### WordPressが向いているケース

本文

### 初心者はどちらから始めるべきか

本文
```

# =========================
# Internal Link Output Rule
# =========================

## 目的

5記事クラスター内の内部リンクを、本文中に自然に挿入する。

## ルール

5記事内部リンク設計を作成した場合でも、WordPress本文貼り付け用には「内部リンク設計表」は入れない。

WordPress本文貼り付け用には、実際に読者がクリックするリンク文だけを自然に入れる。

## 正しい形式

```text
noteで読まれる記事の型を知りたい方は、次の記事も参考になります。

[noteで読まれる記事の書き方を読む](https://jiyoujin.com/note-kakikata-template/)
```

## 禁止形式

```text
書き方を深掘りしたい方は「noteで読まれる記事の書き方」
```

```text
内部リンク設計：
記事1 → 記事2・記事4・記事5
```

```text
関連記事：noteで読まれる記事の書き方
```

```text
こちら
```

# =========================
# Affiliate Link Output Rule
# =========================

## 適用条件

以下の場合のみ、アフィリエイトリンクを本文中に挿入する。

・ユーザーがアフィリエイトリンクを入力した場合  
・アフィリエイトサービスが入力され、仮リンクを使用する場合  

アフィリエイトリンクがない記事では、このルールは適用しない。

---

## 広告・PR表記

アフィリエイトリンクがある場合は、WordPress本文貼り付け用の冒頭に必ず以下のいずれかを入れる。

```text
※本記事には広告・PRが含まれます。
```

または

```text
※この記事はアフィリエイト広告を利用しています。
```

広告・PR表記は、タイトル直前またはタイトル直後に入れる。

---

## 正しい本文内リンク形式

```text
導入前にサービス内容や料金体系を確認したい方は、公式情報を確認してから判断すると安心です。

[サービス名の詳細を確認する](AFFILIATE_LINK_サービス名)
```

または

```text
導入前にサービス内容や料金体系を確認したい方は、公式情報を確認してから判断すると安心です。

[サービス名の詳細を確認する](https://example.com/affiliate-link)
```

---

## 禁止形式

```text
サービス名の詳細を確認する（AFFILIATE_LINK_サービス名）
```

```text
本文内アフィリエイトリンク挿入数：2本
```

だけを本文に書き、実際の本文中にリンクがない状態は禁止。

---

# =========================
# No Affiliate Article Rule
# =========================

## アフィリエイトなし記事のルール

アフィリエイトサービスやアフィリエイトリンクがない記事では、WordPress本文貼り付け用に以下を入れない。

・広告・PR表記  
・AFFILIATE_LINK_〇〇  
・購入を強く促すCTA  
・使用したアフィリエイトサービス  
・本文内アフィリエイトリンク挿入数  
・アフィリエイト導線  

これらは必要な場合、管理情報用にのみ出力する。

## 基本形式

```markdown

パーマリンク：

記事内画像プロンプト：

5記事分のパーマリンク案：

5記事分のURL案：

5記事内部リンク設計：

本文内に挿入した内部リンク：

## アフィリエイト情報

使用したアフィリエイトサービス：

使用したアフィリエイトリンク：
```

管理情報用はWordPress本文に貼り付けない。

# =========================
# Final Output Structure
# =========================

完成記事は必ず以下の順番で出力する。

```text
【1】WordPress本文貼り付け用（.txt）

記事本文

【2】管理情報用

メタ情報・画像情報・内部リンク設計・アフィリエイト情報・CTA案・記事スコア
```

# =========================
# WordPress Body Completion Rule
# =========================

以下のいずれかに該当する場合は、WordPress本文貼り付け用として出力しない。

・WordPress本文貼り付け用が .md ファイルとして扱われている  
・WordPress本文貼り付け用がコードブロックで囲まれている  
・5記事テーマ案が本文に混ざっている  
・5記事パーマリンク案が本文に混ざっている  
・5記事URL案が本文に混ざっている  
・5記事内部リンク設計が本文に混ざっている  
・メタディスクリプションが本文に混ざっている  
・タグが本文に混ざっている  
・画像プロンプトが本文に混ざっている  
・記事スコアが本文に混ざっている  
・アフィリエイト管理情報が本文に混ざっている  
・Markdown表が使われている  
・縦線 `|` を使った表がある  
・タブ区切りの表現がある  
・WordPressで表として挿入されやすい整列表現がある  

---

# =========================
# Final Completion Rule
# =========================

以下の条件を満たしている場合のみ、完成版として出力する。

・WordPress本文貼り付け用と管理情報用が分離されている  
・WordPress本文貼り付け用が .txt のプレーンテキストとして扱われている  
・WordPress本文貼り付け用に読者向け本文だけが入っている  
・管理情報用にメタ情報や記事スコアなどが入っている  
・本文内内部リンクが自然に挿入されている  
・アフィリエイトあり記事ではPR表記と本文内リンクがある  
・アフィリエイトなし記事ではPR表記と仮リンクがない  
・表として挿入されやすい形式を使っていない  
・QUALITY_GATE.mdを通過している
