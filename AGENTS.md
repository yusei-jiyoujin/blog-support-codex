# AGENTS.md

## Project
AI Blog Automation System

## Goal
SEO・AIOに強く、WordPressへそのまま貼り付けられる高品質ブログ記事を生成する。

目的は、薄い量産記事ではなく、検索上位を狙える「深くて読みやすい記事」を1記事ずつ丁寧に完成させること。

ユーザーがアフィリエイトサービスを入力した場合、そのサービスをもとに5記事テーマを自動生成し、自然な収益導線を入れた記事を作成する。

5記事作成時は、5記事を1つの関連記事クラスターとして設計し、記事同士を内部リンクで自然につなげる。

アフィリエイトリンクが入力された場合は、広告・PR表記を入れたうえで、本文中に自然な購入・申込導線を挿入する。

どのカテゴリーの記事でも、CATEGORY_RULES.mdを参照し、記事テーマに合った構成へ最適化する。

最終的には、90点台前半で満足せず、95点以上を狙える記事品質を目指す。

# =========================
# Execution Only Rule
# =========================

## 目的
ユーザーが記事作成を依頼した場合、AGENTS.mdの内容を再出力せず、記事作成を実行する。

## ルール

・ユーザーが「記事作成」「アフィリエイトサービス」「5記事」などを入力した場合、AGENTS.md本文を出力しない  
・AGENTS.mdの説明や再掲は禁止  
・必ず記事テーマ生成または記事本文作成に進む  
・設定ファイルの内容を確認したい場合のみ、AGENTS.mdを出力してよい  

## 正しい動作

ユーザー入力：

アフィリエイトサービス：
・Canva Pro

出力：

1. 5記事テーマ案
2. 5記事分のパーマリンク案
3. 5記事内部リンク設計
4. 記事1の完成版
5. 次の記事への案内

## 禁止動作

・AGENTS.md全文を出力する  
・ルール説明だけで終わる  
・記事本文を作成しない

# =========================
# Affiliate Service Theme Auto Generation Rule
# =========================

## 目的
ユーザーがアフィリエイトサービス名を入力した場合、そのサービスをもとにSEO・AIO・収益化に強い5記事テーマを自動生成する。

## 自動処理ルール

ユーザーがアフィリエイトサービスのみを入力した場合、以下の順番で処理する。

1. 入力されたサービス名を整理する
2. 各サービスの特徴を推定する
3. 想定読者を決める
4. 収益化しやすい検索意図を考える
5. SEO向けの5記事テーマを自動生成する
6. 5記事の役割を分ける
7. 5記事分のパーマリンクを先に作成する
8. 5記事内部リンク設計を作成する
9. 記事1から順番に作成する

## 注意

サービス内容が不明確な場合は、断定しない。  
不確かな情報は「一般的には」「想定される使い方として」「説明会で確認したいポイントとして」など慎重に表現する。  
料金、仕様、キャンペーン、機能など変わりやすい情報は断定しない。

# =========================
# Single Affiliate Service Article Expansion Rule
# =========================

## 目的
ユーザーが1つのアフィリエイトサービスのみを入力した場合でも、そのサービスを軸に5記事テーマを自動生成する。

## 入力例

アフィリエイトサービス：
・Canva Pro

## 自動生成ルール

アフィリエイトサービスが1つだけの場合、以下の5つの切り口で記事テーマを作成する。

1. 始め方記事
2. 活用・稼ぎ方記事
3. 比較記事
4. 注意点・失敗回避記事
5. 評価・おすすめ記事

## テーマ例

アフィリエイトサービス：
・Canva Pro

この場合、以下のように5記事テーマを作成する。

① Canva Proの始め方｜初心者でも使いこなす手順  
② Canva Proで副業はできる？収益化の具体例と活用方法  
③ Canva無料版とCanva Proの違いを比較｜副業に必要なのはどっち？  
④ Canva Proで失敗しないための注意点｜初心者がやりがちなミス  
⑤ Canva Proは副業初心者におすすめ？メリット・デメリットを解説  

## ルール

・1つのサービスでも5記事に展開する  
・同じ内容の繰り返しは禁止  
・各記事の検索意図を必ず変える  
・すべての記事で無理に購入誘導しない  
・比較記事では無料版、代替サービス、使わない選択肢も扱う  
・注意点記事ではデメリットや向かない人も書く  
・収益記事だけでなく、情報記事も混ぜる

# =========================
# 5 Article Internal Link Cluster Rule
# =========================

## 目的
5記事を1つの関連記事クラスターとして作成し、SEO評価・読者回遊・収益導線を強化する。

## 基本ルール

・5記事作成時は、記事本文を作成する前に5記事分のタイトル、役割、パーマリンクを先に作成する  
・作成したパーマリンクを使って、5記事同士の内部リンク設計を行う  
・内部リンクURLは原則として https://jiyoujin.com/パーマリンク/ の形式で出力する  
・各記事には同じ5記事内から2〜3本の内部リンクを自然に挿入する  
・すべての記事にすべての記事を無理にリンクしない  
・検索意図が近い記事同士を優先してつなぐ  
・記事1は中心記事として、記事2〜5へ案内する  
・記事2〜5は必要に応じて記事1へ戻す  
・アンカーテキストは「こちら」だけにしない  
・読者の次の行動が自然に分かる文章でリンクする  
・内部リンクは本文中に自然に挿入し、記事末尾にも一覧として出力する  
・既存記事とのパーマリンク重複は、投稿前にユーザーがWordPress側で確認する前提にする  

## 5記事の役割例

記事1：始め方・全体像  
記事2：稼ぎ方・活用法  
記事3：比較  
記事4：注意点・失敗回避  
記事5：レビュー・おすすめ判断  

## 内部リンク設計例

記事1 → 記事2、記事4  
記事2 → 記事1、記事3、記事4  
記事3 → 記事1、記事5  
記事4 → 記事1、記事2  
記事5 → 記事1、記事3  

## 内部リンク文の例

# =========================
# Permalink Generation Rule
# =========================

## 目的
SEOに強く、WordPressで使いやすいパーマリンクを作成する。

## ルール

・パーマリンクは英語スラッグで作成する  
・短く、意味が分かりやすい形にする  
・メインキーワードを自然に含める  
・日本語、空白、記号は使わない  
・単語はハイフンで区切る  
・長すぎるパーマリンクは禁止  
・数字だけの雑な追記は禁止  
・記事内容とズレたパーマリンクは禁止  
・5記事作成時は、5記事分のパーマリンクを先に作る  
・URL形式は https://jiyoujin.com/slug/ として出力する  
・既存記事との重複は、投稿前にWordPress側で確認する  

## 例

記事タイトル：
A8.netの始め方｜副業ブログ初心者が0→1達成する手順

パーマリンク：
a8net-start-guide-beginner

URL：
https://jiyoujin.com/a8net-start-guide-beginner/

# =========================
# Category Adaptation Rule
# =========================

## 目的
どのカテゴリーの記事でも、テーマに合った構成・深さ・読者ニーズに最適化する。

## ルール

記事作成前に、必ず記事テーマのカテゴリーを判定する。  
カテゴリーごとの必要要素は CATEGORY_RULES.md を参照する。  
すべての記事に同じ構成を無理に当てはめない。  
記事テーマに合わない収益導線や案件例は入れすぎない。  
カテゴリーに合ったH2/H3構成を作成する。  

## カテゴリー判定

以下のいずれか、または近いカテゴリーに分類する。

・副業・収益化系  
・ブログ運営・SEO系  
・アフィリエイト・ASP系  
・AI・ツール活用系  
・旅行・観光・ワーケーション系  
・暮らし・ライフハック系  
・自己投資・スキルアップ系  
・商品レビュー・サービス比較系  
・地域情報・グルメ系  
・健康・美容系  
・ニュース・トレンド解説系  

## 共通維持ルール

どのカテゴリーでも以下は維持する。

・導入文  
・結論  
・要約  
・H2/H3の深掘り  
・具体例  
・失敗例または注意点  
・メリット・デメリット  
・Q&A 5個以上  
・画像用短縮タイトル  
・タイトル入りアイキャッチ画像プロンプト  
・記事内画像プロンプト  
・メタディスクリプション  
・タグ  
・パーマリンク  
・5記事内部リンク設計  
・記事スコア

# =========================
# Affiliate Service Usage Rule
# =========================

## 目的
入力されたアフィリエイトサービスを、読者の悩み解決につながる形で自然に紹介する。

## ルール

・読者の悩み解決につながる場合のみ紹介する  
・本文中に自然な導線を最低1か所入れる  
・記事末尾にCTA案として3パターン出力する  
・サービス名を紹介する場合は、メリットだけでなく注意点も簡単に書く  
・アフィリエイトリンクURLはユーザーが後から差し替える前提で、本文ではサービス名のみを使用する  
・アフィリエイトリンクが入力されている場合は、そのURLを本文内に自然に挿入する  

## 禁止事項

・関係ないサービスを無理に入れる  
・「絶対稼げる」「必ず儲かる」などの誇大表現を使う  
・読者の不安を過剰に煽る  
・すべての見出しでサービス紹介をする  
・記事全体を広告記事のようにする  
・根拠のないランキング表現をする  
・アフィリエイトリンクを過剰に入れる  

## 出力ルール

各記事の最後に以下を必ず出力する。

使用したアフィリエイトサービス：  
使用したアフィリエイトリンク：  
本文内リンク挿入数：  
リンク挿入位置：  
アフィリエイト導線：  
CTA案：

# =========================
# Affiliate Link Auto Insert Rule
# =========================

## 目的
アフィリエイト記事で、読者の悩み解決に沿った自然な購入・申込導線を作る。

## ルール

・ユーザーがアフィリエイトリンクを入力した場合、記事本文内に自然に挿入する  
・リンクURLが入力されていない場合は、仮リンクとして「AFFILIATE_LINK_サービス名」を使用する  
・アフィリエイトリンクは1記事あたり2〜3か所までを目安にする  
・リンクを入れすぎて広告っぽくしない  
・本文中のリンクは、読者の悩みが解決される流れの中で挿入する  
・比較、注意点、向いている人、まとめ、CTAのいずれかに自然に入れる  
・「今すぐ買うべき」「絶対おすすめ」などの強い押し売り表現は禁止  
・読者が自分で判断できるように、メリットだけでなく注意点も書く  
・料金、キャンペーン、成果、効果など変わりやすい情報は断定しない  
・リンクのアンカーテキストは自然にする  
・「こちら」だけのリンク文は禁止する  

## 広告表記ルール

・アフィリエイトリンクを含む記事では、記事冒頭に広告・PR表記を入れる  
・広告表記は読者が分かりやすい位置に表示する  
・小さすぎる表記、分かりにくい表現、記事末尾だけの表記は避ける  

## 広告表記例

※本記事には広告・PRが含まれます。

または

※この記事はアフィリエイト広告を利用しています。

## リンク挿入位置

以下の場所に自然に挿入する。

1. 導入後または結論後の軽い案内  
2. メリット・向いている人・比較パートの後  
3. まとめ前またはCTA部分  

## リンク挿入例

AI学習を独学で進めるか、スクールで相談しながら進めるか迷っている方は、まず個別説明会で学習内容やサポート体制を確認しておくと判断しやすくなります。

[サービス名の詳細を確認する](アフィリエイトリンク)

# =========================
# Sequential Article Generation Rule
# =========================

## 目的
5記事を一括で浅く作らず、1記事ずつ順番に作成・評価・改善して完成させる。

## ルール

ユーザーが5記事分のテーマ、またはアフィリエイトサービスを入力した場合、以下の流れで処理する。

1. 5記事全体のテーマを確認、または自動生成する
2. 5記事分の役割を決める
3. 5記事分のパーマリンクを作成する
4. 5記事内部リンク設計を作成する
5. 記事1だけを作成する
6. 記事1をSEO/AIO/網羅性/読みやすさ/収益性の観点で評価する
7. 95点未満なら改善する
8. 記事1の完成版をコピー用BOX形式で出力する
9. 記事1の最後に「続き」と入力すれば記事2を作成できることを案内する
10. ユーザーが「続き」と入力したら記事2を作成する
11. 同じ流れで記事5まで進める

## 禁止事項

・5記事を一括で本文出力しない  
・3記事＋2記事に分けて本文出力しない  
・文字数を削って複数記事をまとめない  
・評価前の記事を完成版として出力しない  
・記事スコア95点未満の記事を完成扱いしない  
・メタ情報、タグ、パーマリンク、画像プロンプト、内部リンク案、CTA案を省略しない  
・1記事が完成する前に次の記事へ進まない

# =========================
# H3 Depth Rule
# =========================

## 目的
H3見出し直後の本文が薄くなることを防ぎ、SEO上位を狙える深い記事にする。

## ルール

・H3見出しの直後を一言、1文、箇条書きだけで終わらせない  
・各H3には最低150〜200文字以上の本文を入れる  
・重要なH3では250〜400文字を目安にする  
・H3本文では、結論・理由・具体例・注意点または改善策を入れる  
・箇条書きを使う場合も、その前後に必ず補足説明を入れる  
・「理由：」「改善：」だけの短い説明で終わらせない  
・H3ごとに読者の疑問が1つ解決する状態にする  
・H3の内容が薄い場合は、読者の不安、具体例、実践手順、注意点を追加して補強する  

## H3本文の基本構成

### H3見出し
1. まず結論を書く  
2. なぜそうなのか理由を書く  
3. 初心者にもわかる具体例を書く  
4. 注意点または改善策を書く  
5. 必要なら次の行動を示す
