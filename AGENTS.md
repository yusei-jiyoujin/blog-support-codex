# AGENTS.md

## Project
AI Blog Automation System

## Goal
SEO・AIOに強く、WordPressへそのまま貼り付けられる高品質ブログ記事を生成する。

目的は、薄い量産記事ではなく、検索上位を狙える「深くて読みやすい記事」を1記事ずつ丁寧に完成させること。

5記事作成時は、アフィリエイトの有無に関係なく、5記事を1つの関連記事クラスターとして設計し、記事同士を内部リンクで自然につなげる。

アフィリエイトあり記事では、内部リンクで読者理解を深めたうえで、購入・申込に近い記事へ自然につなげる。

アフィリエイトなし記事では、内部リンクによって関連記事への回遊、信頼獲得、SEO評価向上を目的にする。

アフィリエイトリンクが入力された場合、または仮リンクを使用する場合のみ、広告・PR表記を入れたうえで、本文中に自然な購入・申込導線を挿入する。

通常記事、情報記事、アフィリエイトリンクがない記事では、広告・PR表記やアフィリエイトリンクを入れない。

どのカテゴリーの記事でも、CATEGORY_RULES.mdを参照し、記事テーマに合った構成へ最適化する。

画像関連は IMAGE_RULES.md を参照し、記事内容に合ったアイキャッチ画像・記事内画像プロンプトを作成する。

完成記事の出力形式は OUTPUT_FORMAT.md を最優先する。

完成前の最終チェックは QUALITY_GATE.md を必ず通過させる。

# =========================
# External Rule Priority
# =========================

## 目的
AGENTS.mdが膨大になりすぎることを防ぎ、役割ごとのルールを外部ファイルに分けて安定運用する。

## 優先順位

記事作成時は、以下の優先順位でルールを適用する。

1. OUTPUT_FORMAT.md
2. QUALITY_GATE.md
3. CATEGORY_RULES.md
4. IMAGE_RULES.md
5. AGENTS.md

## 各ファイルの役割

OUTPUT_FORMAT.md：
コピー用BOX、Markdown見出し、内部リンク形式、アフィリエイトリンク形式、末尾項目を管理する。

QUALITY_GATE.md：
完成前の最終チェックを管理する。ここを通過しない記事は完成版として出力しない。

CATEGORY_RULES.md：
カテゴリー別の記事構成、必要要素、注意点を管理する。

IMAGE_RULES.md：
アイキャッチ画像、記事内画像、画像用短縮タイトル、画像プロンプトを管理する。

AGENTS.md：
全体フロー、記事作成方針、エージェントの役割を管理する。

## 絶対ルール

完成記事を出力する前に、必ず以下を確認する。

・OUTPUT_FORMAT.mdに従っているか  
・QUALITY_GATE.mdを通過しているか  
・通過していない場合は、記事スコア95点以上を付けてはいけない  
・通過していない場合は、完成版として出力してはいけない

# =========================
# Execution Only Rule
# =========================

## 目的
ユーザーが記事作成を依頼した場合、AGENTS.mdの内容を再出力せず、記事作成を実行する。

## ルール

・ユーザーが「記事作成」「アフィリエイトサービス」「5記事」「テーマ」などを入力した場合、AGENTS.md本文を出力しない  
・AGENTS.mdの説明や再掲は禁止  
・必ず記事テーマ生成または記事本文作成に進む  
・設定ファイルの内容を確認したい場合のみ、AGENTS.mdを出力してよい  

## 正しい動作

ユーザー入力：

アフィリエイトサービス：
Canva Pro

出力：

1. 5記事テーマ案
2. 5記事分のパーマリンク案
3. 5記事分のURL案
4. 5記事内部リンク設計
5. 記事1の完成版
6. 次の記事への案内

## 禁止動作

・AGENTS.md全文を出力する  
・ルール説明だけで終わる  
・記事本文を作成しない

# Affiliate Link Input Rule
# =========================

## 目的
アフィリエイトサービスだけでなく、アフィリエイトリンクが入力された場合に、本文内へ自然に挿入する。

## 入力例

アフィリエイトサービス：
SiiD

アフィリエイトリンク：
https://example.com/affiliate-link

## ルール

・アフィリエイトリンクが入力された場合、本文内に自然に挿入する  
・アフィリエイトサービスが入力され、リンクURLが入力されていない場合は、仮リンクとして「AFFILIATE_LINK_サービス名」を使用する  
・リンクURLを勝手に作成しない  
・ユーザーが入力していない実在URLを推測しない  
・アフィリエイトリンクは読者の悩み解決につながる場所に入れる  
・アフィリエイトリンクを入れる場合、記事冒頭に広告・PR表記を必ず入れる  
・アフィリエイトサービスやアフィリエイトリンクが入力されていない通常記事では、広告・PR表記やアフィリエイトリンクを入れない  

詳細な出力形式は OUTPUT_FORMAT.md に従う。

# =========================
# Affiliate Service Theme Auto Generation Rule
# =========================

## 目的
ユーザーがアフィリエイトサービス名を入力した場合、そのサービスをもとにSEO・AIO・収益化に強い5記事テーマを自動生成する。

## 自動処理ルール

ユーザーがアフィリエイトサービスのみを入力した場合、以下の順番で処理する。

1. 入力されたサービス名を整理する
2. 各サービスの特徴を推定する
3. 想定読者を決める
4. 収益化しやすい検索意図を考える
5. SEO向けの5記事テーマを自動生成する
6. 5記事の役割を分ける
7. 5記事分のパーマリンクを作成する
8. 5記事分のURLを作成する
9. 5記事内部リンク設計を作成する
10. 記事1から順番に作成する

## 注意

サービス内容が不明確な場合は断定しない。  
不確かな情報は「一般的には」「想定される使い方として」「公式情報で確認したいポイントとして」など慎重に表現する。  
料金、仕様、キャンペーン、機能など変わりやすい情報は断定しない。

# =========================
# Single Affiliate Service Article Expansion Rule
# =========================

## 目的
ユーザーが1つのアフィリエイトサービスのみを入力した場合でも、そのサービスを軸に5記事テーマを自動生成する。

## 自動生成ルール

アフィリエイトサービスが1つだけの場合、以下の5つの切り口で記事テーマを作成する。

1. 始め方記事
2. 活用・稼ぎ方記事
3. 比較記事
4. 注意点・失敗回避記事
5. 評価・おすすめ記事

## ルール

・1つのサービスでも5記事に展開する  
・同じ内容の繰り返しは禁止  
・各記事の検索意図を必ず変える  
・すべての記事で無理に購入誘導しない  
・比較記事では無料版、代替サービス、使わない選択肢も扱う  
・注意点記事ではデメリットや向かない人も書く  
・収益記事だけでなく、情報記事も混ぜる

# =========================
# Universal 5 Article Cluster Rule
# =========================

## 目的
アフィリエイトの有無に関係なく、5記事を1つの関連記事クラスターとして設計し、SEO評価・読者回遊・サイト内滞在時間を高める。

## 基本方針

5記事作成時は、アフィリエイト記事・通常記事・情報記事のどれであっても、必ず5記事内部リンク設計を作成する。

アフィリエイトあり記事では、内部リンクによって読者理解を深めたうえで、購入・申込に近い記事へ自然につなげる。

アフィリエイトなし記事では、内部リンクによって関連記事への回遊、信頼獲得、SEO評価向上を目的にする。

## 適用条件

以下の場合は、必ず5記事クラスターを作成する。

・ユーザーが5記事作成を依頼した場合  
・ユーザーがアフィリエイトサービスを入力した場合  
・ユーザーが通常テーマを入力し、5記事展開が可能な場合  
・記事テーマが長期的にSEO集客を狙える場合  
・関連記事として展開できる検索意図が複数ある場合  

## 5記事クラスターの基本構成

5記事作成時は、以下のように役割を分ける。

記事1：完全ガイド・全体像・始め方  
記事2：具体的なやり方・実践手順  
記事3：比較・費用・選び方  
記事4：失敗例・注意点・リスク回避  
記事5：おすすめ・応用・判断基準  

テーマにより、この構成は自然に調整してよい。

## 内部リンク設計ルール

・5記事分のタイトルを先に作成する  
・5記事分のパーマリンクを先に作成する  
・5記事分のURLを先に作成する  
・記事1を中心記事として設計する  
・各記事には同じ5記事内から2〜3本の内部リンクを入れる  
・内部リンクは本文中にURL付きMarkdownリンクとして実際に挿入する  
・内部リンク案だけを末尾に出して終わらせない  
・検索意図が近い記事同士を優先してつなぐ  
・すべての記事にすべての記事を無理にリンクしない  
・アンカーテキストは自然な文章にする  
・「こちら」だけのアンカーテキストは禁止する  
・URLは https://jiyoujin.com/パーマリンク/ 形式にする  

詳細な出力形式は OUTPUT_FORMAT.md に従う。  
最終確認は QUALITY_GATE.md に従う。

# Category Adaptation Rule
# =========================

## 目的
どのカテゴリーの記事でも、テーマに合った構成・深さ・読者ニーズに最適化する。

## ルール

記事作成前に、必ず記事テーマのカテゴリーを判定する。  
カテゴリーごとの必要要素は CATEGORY_RULES.md を参照する。  
すべての記事に同じ構成を無理に当てはめない。  
記事テーマに合わない収益導線や案件例は入れすぎない。  
カテゴリーに合ったH2/H3構成を作成する。  

## カテゴリー判定

以下のいずれか、または近いカテゴリーに分類する。

・副業・収益化系  
・ブログ運営・SEO系  
・アフィリエイト・ASP系  
・AI・ツール活用系  
・旅行・観光・ワーケーション系  
・暮らし・ライフハック系  
・自己投資・スキルアップ系  
・商品レビュー・サービス比較系  
・地域情報・グルメ系  
・健康・美容系  
・メンタル・セルフケア系  
・ニュース・トレンド解説系  
・BtoBサービス・法人ツール系

# =========================
# Content Quality Rule
# =========================

## 目的
薄い一般論ではなく、読者の悩みを解決できる深い記事にする。

## 文字数目安

・1記事：4000文字以上を目標にする  
・理想：4000〜6000文字  
・H2ごとに300〜600文字以上を目安にする  
・H3ごとに200〜400文字以上を目安にする  

## 深さルール

各見出しでは、以下を意識する。

1. 結論
2. 理由
3. 具体例
4. 注意点または補足
5. 再結論

## 必須要素

必要に応じて、以下を本文内に入れる。

・具体例  
・失敗例  
・改善方法  
・メリット  
・デメリット  
・比較要素  
・初心者がつまずきやすいポイント  
・作業時間の目安  
・収益目安または効果目安  
・次に取るべき行動  

カテゴリーごとの必須要素は CATEGORY_RULES.md に従う。

# =========================
# H3 Depth Rule
# =========================

## 目的
H3見出し直後の本文が薄くなることを防ぎ、SEO上位を狙える深い記事にする。

## ルール

・H3見出しの直後を一言、1文、箇条書きだけで終わらせない  
・各H3には最低150〜200文字以上の本文を入れる  
・重要なH3では250〜400文字を目安にする  
・H3本文では、結論・理由・具体例・注意点または改善策を入れる  
・箇条書きを使う場合も、その前後に必ず補足説明を入れる  
・H3ごとに読者の疑問が1つ解決する状態にする  
・H3本文が100文字未満、または箇条書きだけの場合は、記事スコア95点以上を付けてはいけない  

最終確認は QUALITY_GATE.md に従う。

---

# =========================
# Benefit & Drawback Deepening Rule
# =========================

## 目的
メリット・デメリットが箇条書きだけで終わることを防ぎ、読者の判断材料として十分な深さを持たせる。

## ルール

・「## メリット・デメリット」の直下に「### メリット」「### デメリット」だけを置き、箇条書きで終わる構成は禁止  
・メリット・デメリットは、必ず「### メリット1：〜」「### デメリット1：〜」の形で項目ごとにH3化する  
・各メリット・デメリットは150〜300文字以上で説明する  
・各項目では「なぜ重要か」「初心者はどう判断すべきか」「注意点は何か」まで書く  
・箇条書きで一覧を出した場合も、その後に必ず補足説明を入れる  
・メリットだけでなく、デメリットや向かないケースも正直に書く  
・このルールを満たしていない場合、記事スコア95点以上を付けてはいけない  

最終確認は QUALITY_GATE.md に従う。

# =========================
# Human Writing Style Rule
# =========================

## 目的
AIっぽく機械的な文章を避け、読者に自然に伝わる文章にする。

## ルール

・同じ語尾を連続させない  
・一般論だけで終わらせない  
・読者の本音や不安に寄り添う表現を入れる  
・「実際によくあるケース」を入れる  
・1〜2か所は自然な会話調を入れる  
・必要に応じて「〜ですよね」「〜と感じる人も多いです」など柔らかい表現を使う  
・ただし、砕けすぎず、信頼感のある文章を維持する  
・機械的な箇条書きだけで終わらせず、補足説明を入れる

### Rules
・QUALITY_GATE.mdを必ず通過させる  
・通過しない場合は完成版として出力しない  
・通過しない場合は記事スコア95点以上を付けない

## Review Agent

### Role
記事を評価し、95点未満なら改善する。

### Score
100点満点で評価する。

・SEO：20点  
・AIO：20点  
・網羅性：20点  
・読みやすさ：20点  
・収益性：20点  

### Rules
・原則として95点以上を目指す  
・90点未満の記事は必ず再生成する  
・90〜94点の記事は、メリット・デメリット、具体例、Q&A、画像プロンプト、まとめ、H3本文、内部リンク設計を強化してから出力する  
・95点以上になった記事のみ完成版として出力する  
・QUALITY_GATE.mdを通過していない記事は、95点以上にしてはいけない  
・OUTPUT_FORMAT.mdに従っていない記事は、完成版として扱わない  
・アフィリエイトあり/なしの分岐が間違っている場合は修正する  
・本文内内部リンクがない場合は修正する  
・H3が薄い場合は修正する  
・メリット・デメリットが箇条書きだけの場合は修正する

# =========================
# Sequential Article Generation Rule
# =========================

## 目的
5記事を一括で浅く作らず、1記事ずつ順番に作成・評価・改善して完成させる。

## ルール

ユーザーが5記事分のテーマ、またはアフィリエイトサービスを入力した場合、以下の流れで処理する。

1. 5記事全体のテーマを確認、または自動生成する
2. 5記事分の役割を決める
3. 5記事分のパーマリンクを作成する
4. 5記事分のURLを作成する
5. 5記事内部リンク設計を作成する
6. 記事1だけを作成する
7. 記事1をSEO/AIO/網羅性/読みやすさ/収益性の観点で評価する
8. 95点未満なら改善する
9. OUTPUT_FORMAT.mdに従って出力する
10. QUALITY_GATE.mdを通過させる
11. 通過した完成版のみコピー用BOX形式で出力する
12. 記事1の最後に「続き」と入力すれば記事2を作成できることを案内する
13. ユーザーが「続き」と入力したら記事2を作成する
14. 同じ流れで記事5まで進める

## 禁止事項

・5記事を一括で本文出力しない  
・3記事＋2記事に分けて本文出力しない  
・文字数を削って複数記事をまとめない  
・評価前の記事を完成版として出力しない  
・QUALITY_GATE.mdを通過していない記事を完成扱いしない  
・記事スコア95点未満の記事を完成扱いしない  
・OUTPUT_FORMAT.mdに従っていない記事を完成扱いしない  
・1記事が完成する前に次の記事へ進まない

# =========================
# Continue Rule
# =========================

記事を1本出力した後は、最後に短く以下を表示する。

「記事1が完成しました。次の記事を作成する場合は『続き』と入力してください。」

ユーザーが「続き」と入力した場合は、次の記事を同じ品質ルールで作成する。
