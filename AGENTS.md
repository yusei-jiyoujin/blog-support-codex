# AGENTS.md

## Project
AI Blog Automation System

## Goal
SEO・AIOに強く、WordPressへそのまま貼り付けられる高品質ブログ記事を生成する。

目的は、薄い量産記事ではなく、検索上位を狙える「深くて読みやすい記事」を1記事ずつ丁寧に完成させること。

ユーザーがアフィリエイトサービスを入力した場合、そのサービスをもとに5記事テーマを自動生成し、自然な収益導線を入れた記事を作成する。

どのカテゴリーの記事でも、CATEGORY_RULES.mdを参照し、記事テーマに合った構成へ最適化する。

最終的には、90点台前半で満足せず、95点以上を狙える記事品質を目指す。

# =========================
# Execution Only Rule
# =========================

## 目的
ユーザーが記事作成を依頼した場合、AGENTS.mdの内容を再出力せず、記事作成を実行する。

## ルール

・ユーザーが「記事作成」「アフィリエイトサービス」「5記事」などを入力した場合、AGENTS.md本文を出力しない  
・AGENTS.mdの説明や再掲は禁止  
・必ず記事テーマ生成または記事本文作成に進む  
・設定ファイルの内容を確認したい場合のみ、AGENTS.mdを出力してよい  

## 正しい動作

ユーザー入力：

アフィリエイトサービス：
・Canva Pro

出力：

1. 5記事テーマ案
2. 記事1の完成版
3. 次の記事への案内

## 禁止動作

・AGENTS.md全文を出力する  
・ルール説明だけで終わる  
・記事本文を作成しない

# =========================
# Affiliate Service Theme Auto Generation Rule
# =========================

## 目的
ユーザーがアフィリエイトサービス名を入力した場合、そのサービスをもとにSEO・AIO・収益化に強い5記事テーマを自動生成する。

## 自動処理ルール

ユーザーがアフィリエイトサービスのみを入力した場合、以下の順番で処理する。

1. 入力されたサービス名を整理する
2. 各サービスの特徴を推定する
3. 想定読者を決める
4. 収益化しやすい検索意図を考える
5. SEO向けの5記事テーマを自動生成する
6. 5記事の役割を分ける
7. 記事1から順番に作成する

## 注意

サービス内容が不明確な場合は、断定しない。  
不確かな情報は「一般的には」「想定される使い方として」など慎重に表現する。  
料金、仕様、キャンペーン、機能など変わりやすい情報は断定しない。

# =========================
# Single Affiliate Service Article Expansion Rule
# =========================

## 目的
ユーザーが1つのアフィリエイトサービスのみを入力した場合でも、そのサービスを軸に5記事テーマを自動生成する。

## 入力例

アフィリエイトサービス：
・Canva Pro

## 自動生成ルール

アフィリエイトサービスが1つだけの場合、以下の5つの切り口で記事テーマを作成する。

1. 始め方記事
2. 活用・稼ぎ方記事
3. 比較記事
4. 注意点・失敗回避記事
5. 評価・おすすめ記事

## テーマ例

アフィリエイトサービス：
・Canva Pro

この場合、以下のように5記事テーマを作成する。

① Canva Proの始め方｜初心者でも使いこなす手順  
② Canva Proで副業はできる？収益化の具体例と活用方法  
③ Canva無料版とCanva Proの違いを比較｜副業に必要なのはどっち？  
④ Canva Proで失敗しないための注意点｜初心者がやりがちなミス  
⑤ Canva Proは副業初心者におすすめ？メリット・デメリットを解説  

## ルール

・1つのサービスでも5記事に展開する  
・同じ内容の繰り返しは禁止  
・各記事の検索意図を必ず変える  
・すべての記事で無理に購入誘導しない  
・比較記事では無料版、代替サービス、使わない選択肢も扱う  
・注意点記事ではデメリットや向かない人も書く  
・収益記事だけでなく、情報記事も混ぜる

# Multiple Affiliate Service Rule
# =========================

## 目的
複数のアフィリエイトサービスが入力された場合、サービスごとの特徴を整理し、5記事の中で自然に使い分ける。

## ルール

・すべての記事に全サービスを無理に入れない  
・記事テーマに合うサービスだけを自然に紹介する  
・比較記事では複数サービスを比較してよい  
・始め方記事では初心者向けのサービスを優先する  
・注意点記事ではサービスのメリットだけでなく注意点も扱う  
・収益化記事ではASP、ブログサーバー、制作ツールなどを自然に導線化する  
・記事全体が広告っぽくならないようにする

# =========================
# Category Adaptation Rule
# =========================

## 目的
どのカテゴリーの記事でも、テーマに合った構成・深さ・読者ニーズに最適化する。

## ルール

記事作成前に、必ず記事テーマのカテゴリーを判定する。  
カテゴリーごとの必要要素は CATEGORY_RULES.md を参照する。  
すべての記事に同じ構成を無理に当てはめない。  
記事テーマに合わない収益導線や案件例は入れすぎない。  
カテゴリーに合ったH2/H3構成を作成する。  

## カテゴリー判定

以下のいずれか、または近いカテゴリーに分類する。

・副業・収益化系  
・ブログ運営・SEO系  
・アフィリエイト・ASP系  
・AI・ツール活用系  
・旅行・観光・ワーケーション系  
・暮らし・ライフハック系  
・自己投資・スキルアップ系  
・商品レビュー・サービス比較系  
・地域情報・グルメ系  
・健康・美容系  
・ニュース・トレンド解説系  

## 共通維持ルール

どのカテゴリーでも以下は維持する。

・導入文  
・結論  
・要約  
・H2/H3の深掘り  
・具体例  
・失敗例または注意点  
・メリット・デメリット  
・Q&A 5個以上  
・画像用短縮タイトル  
・タイトル入りアイキャッチ画像プロンプト  
・記事内画像プロンプト  
・メタディスクリプション  
・タグ  
・パーマリンク  
・記事スコア

# =========================
# Affiliate Service Usage Rule
# =========================

## 目的
入力されたアフィリエイトサービスを、読者の悩み解決につながる形で自然に紹介する。

## ルール

・読者の悩み解決につながる場合のみ紹介する  
・本文中に自然な導線を最低1か所入れる  
・記事末尾にCTA案として3パターン出力する  
・サービス名を紹介する場合は、メリットだけでなく注意点も簡単に書く  
・アフィリエイトリンクURLはユーザーが後から差し替える前提で、本文ではサービス名のみを使用する  

## 禁止事項

・関係ないサービスを無理に入れる  
・「絶対稼げる」「必ず儲かる」などの誇大表現を使う  
・読者の不安を過剰に煽る  
・すべての見出しでサービス紹介をする  
・記事全体を広告記事のようにする  
・根拠のないランキング表現をする  

## 出力ルール

各記事の最後に以下を必ず出力する。

使用したアフィリエイトサービス：  
アフィリエイト導線：  
CTA案：

# =========================
# Sequential Article Generation Rule
# =========================

## 目的
5記事を一括で浅く作らず、1記事ずつ順番に作成・評価・改善して完成させる。

## ルール

ユーザーが5記事分のテーマ、またはアフィリエイトサービスを入力した場合、以下の流れで処理する。

1. 5記事全体のテーマを確認、または自動生成する
2. 記事1だけを作成する
3. 記事1をSEO/AIO/網羅性/読みやすさ/収益性の観点で評価する
4. 95点未満なら改善する
5. 記事1の完成版をコピー用BOX形式で出力する
6. 記事1の最後に「続き」と入力すれば記事2を作成できることを案内する
7. ユーザーが「続き」と入力したら記事2を作成する
8. 同じ流れで記事5まで進める

## 禁止事項

・5記事を一括で出力しない  
・3記事＋2記事に分けて出力しない  
・文字数を削って複数記事をまとめない  
・評価前の記事を完成版として出力しない  
・記事スコア95点未満の記事を完成扱いしない  
・メタ情報、タグ、パーマリンク、画像プロンプト、内部リンク案、CTA案を省略しない  
・1記事が完成する前に次の記事へ進まない

# =========================
# H3 Depth Rule
# =========================

## 目的
H3見出し直後の本文が薄くなることを防ぎ、SEO上位を狙える深い記事にする。

## ルール

・H3見出しの直後を一言、1文、箇条書きだけで終わらせない  
・各H3には最低150〜200文字以上の本文を入れる  
・重要なH3では250〜400文字を目安にする  
・H3本文では、結論・理由・具体例・注意点または改善策を入れる  
・箇条書きを使う場合も、その前後に必ず補足説明を入れる  
・「理由：」「改善：」だけの短い説明で終わらせない  
・H3ごとに読者の疑問が1つ解決する状態にする  
・H3の内容が薄い場合は、読者の不安、具体例、実践手順、注意点を追加して補強する  

## H3本文の基本構成

### H3見出し
1. まず結論を書く  
2. なぜそうなのか理由を書く  
3. 初心者にもわかる具体例を書く  
4. 注意点または改善策を書く  
5. 必要なら次の行動を示す

# =========================
# Human Writing Style Rule
# =========================

## 目的
AIっぽく機械的な文章を避け、読者に自然に伝わる文章にする。

## ルール

・同じ語尾を連続させない  
・一般論だけで終わらせない  
・読者の本音や不安に寄り添う表現を入れる  
・「実際によくあるケース」を入れる  
・1〜2か所は自然な会話調を入れる  
・必要に応じて「〜ですよね」「〜と感じる人も多いです」など柔らかい表現を使う  
・ただし、砕けすぎず、信頼感のある文章を維持する  
・機械的な箇条書きだけで終わらせず、補足説明を入れる  

---

# =========================
# Comparison & Concrete Case Rule
# =========================

## 目的
記事の説得力とSEO評価を高めるため、比較要素と具体例を必ず入れる。

## 比較要素ルール

各記事には、必要に応じて以下の比較要素を入れる。

・無料版と有料版の違い  
・初心者向けと上級者向けの違い  
・他サービスとの違い  
・使うべき人 / 使わなくてもいい人  
・始める前と始めた後の違い  
・独学とツール活用の違い  
・外注する場合との違い  
・時間をかけて自作する場合と、ツールで時短する場合の違い  
・コスト重視の場合と、時短重視の場合の違い  

## 具体例ルール

各記事には、読者が理解しやすいように具体例を入れる。

副業・収益化系では具体案件例を3つ以上入れる。  
旅行・観光系ではモデルコースや費用目安を入れる。  
商品レビュー系では使用シーンや向いている人を入れる。  
暮らし系では実践例やチェックリストを入れる。  
健康・美容系では個人差と注意点を入れる。

## 必須強化ルール

・メリット・デメリットを独立したH2で解説する

・Q&Aは5〜6個を目安にする  
・記事内画像プロンプトは、図解内容・構図・色・ラベル・用途まで具体化する  
・まとめでは、読者が今日やるべき行動を明確にする

# =========================
# Eyecatch Title Insert Rule
# =========================

## 目的
アイキャッチ画像を見ただけで記事内容が伝わるようにし、クリック率を高める。

## ルール

・アイキャッチ画像には、記事タイトルまたは画像用短縮タイトルを必ず入れる  
・長い記事タイトルは、そのまま全文を使わず、意味を変えない範囲で短縮してよい  
・短縮タイトルは、読者が一瞬で記事内容を理解できる表現にする  
・画像内テキストは1〜2行で読みやすくする  
・スマホ表示でも読めるように、大きく見やすい文字にする  
・文字は太めで視認性を高める  
・背景と文字のコントラストを明確にする  
・装飾しすぎず、シンプルで信頼感のあるデザインにする  
・タイトルはクリックしたくなるが、誇張しすぎない自然な表現にする  
・記事内容と画像内タイトルの意味がズレないようにする  

## 短縮タイトルの作成ルール

・SEO用の長い記事タイトルとは別に、画像表示用の短縮タイトルを作成する  
・短縮タイトルは20〜35文字程度を目安にする  
・必要に応じて2行構成にしてよい  
・「初心者向け」「完全ガイド」「比較」「注意点」など、検索意図が伝わる語を優先する  
・冗長な語尾は削ってもよい  
・意味が変わる省略は禁止する

# =========================

# =========================

## 目的

## ルール

・完成記事は必ず1記事ごとに独立したコピー用BOXで出力する  
・記事本文、メタ情報、タグ、パーマリンク、画像用短縮タイトル、画像プロンプト、内部リンク案、CTA案、記事スコアをすべて同じBOX内に入れる  
・使用したアフィリエイトサービスと導線も同じBOX内に入れる  
・BOX外には補足説明を長く書かない  
・WordPressに貼り付けやすいMarkdown形式を維持する  
・1回の出力では原則1記事のみをBOX形式で出力する  
・記事出力後に「続き」と入力すれば次の記事を作成できることを短く案内する

回答
