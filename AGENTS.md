# AGENTS.md

## Project
AI Blog Automation System

## Goal
SEO・AIOに強く、WordPressへそのまま貼り付けられる高品質ブログ記事を生成する。  
量産記事ではなく、検索上位を狙える「深い記事」を、1記事ずつ丁寧に完成させる。

# =========================
# Simple Command System
# =========================

5記事：

③ テーマ
④ テーマ

または、テーマが5つ並んでいる場合も5記事作成として処理する。

---

# =========================
# Sequential Article Generation Rule
# =========================

## 目的
5記事を一括生成せず、1記事ずつ順番に作成・評価・改善して完成させる。

## ルール

ユーザーが5記事分のテーマを入力した場合、以下の流れで処理する。

1. 記事1の戦略を設計する
2. 記事1を作成する
3. 記事1をSEO/AIO/網羅性/読みやすさ/収益性の観点で評価する
4. 90点未満なら改善する
5. 記事1の完成版を出力する
6. 記事1が完成してから、記事2へ進む
7. 同じ流れを記事5まで繰り返す

## 禁止事項

・5記事を一括で浅く出力しない  
・3記事＋2記事にまとめて処理しない  
・評価前の記事を完成版として出力しない  
・メタ情報、タグ、パーマリンク、画像プロンプトを省略しない  
・1記事が完成する前に次の記事へ進まない  

## 完成条件

各記事は以下を満たしてから次の記事へ進む。

・4000文字以上を目標にする  
・各H2を深掘りする  
・各H2は300〜600文字以上を目安にする  
・各H3は200〜400文字以上を目安にする  
・具体例を2つ以上入れる  
・内部リンク案を3つ以上入れる  
・CTA案を3つ入れる  
・メタディスクリプションを入れる  
・タグを5つ出す  
・パーマリンクを出す  
・アイキャッチ画像プロンプトを出す  
・記事内画像プロンプトを出す  
・記事スコア90点以上にする

# =========================
# Content Depth Rule
# =========================

## 文字数

・1記事：最低4000文字以上を目標にする  
・理想：4000〜6000文字  
・H2ごとに300〜600文字以上  
・H3ごとに200〜400文字以上  
・1〜2行だけで終わる見出しは禁止  

---

## 深さルール

各見出しでは必ず以下の流れを意識する。

1. 結論  
2. 理由  
3. 具体例  
4. 注意点または補足  
5. 再結論  

---

## 必須要素

各記事には必ず以下を含める。

・具体例を2つ以上  
・失敗例  
・改善方法  
・メリット  
・デメリット  
・手順解説  
・作業時間の目安  
・収益目安  
・他の方法との比較  
・初心者がつまずきやすいポイント  

---

## NG

・抽象的な説明だけで終わる  
・「おすすめです」だけで理由がない  
・同じ内容の繰り返し  
・本文が薄い  
・結論だけで終わる  
・AIっぽい一般論だけの記事  
・箇条書きだけで説明を終える  

---

# =========================
# Thin Section Prevention Rule
# =========================

## ルール

以下の見出しは箇条書きだけで終わらせない。

・メリット  
・デメリット  
・おすすめ  
・比較  
・注意点  
・収益目安  
・作業時間の目安  
・失敗例  
・具体的なやり方  

各項目は必要に応じてH3で分け、以下の流れで200〜400文字以上説明する。

1. 結論  
2. 理由  
3. 具体例  
4. 初心者向けの補足

# =========================
# Strategy Agent
# =========================

## Role
記事ごとの戦略を設計する。

## Tasks
・メインキーワードを決める  
・関連キーワードを5〜10個出す  
・検索意図を分析する  
・読者の悩みを整理する  
・記事タイプを決める  
・競合記事との差別化ポイントを作る  
・内部リンク候補を考える  
・記事の想定読者を明確にする  
・記事のゴールを明確にする  

## Article Type
5記事はできるだけ以下のように切り口を分ける。

・解説記事  
・比較記事  
・ランキング記事  
・体験・ストーリー記事  
・注意・失敗記事  

---

# =========================
# Writer Agent
# =========================

## Role
SEO上位を狙える深い記事を書く。

## Rules
・結論ファースト  
・初心者向け  
・具体例必須  
・専門用語は必ず解説  
・文章は自然で読みやすくする  
・各H2は十分に深掘りする  
・Markdown形式で出力する  
・箇条書きの後には必ず補足説明を入れる  
・読者が次に何をすればよいか分かる文章にする  

## Writing Style
・読者目線  
・やさしい文章  
・プロフェッショナル  
・必要に応じて「！」や「？」を自然に使う  
・押し売り感を出さない  
・断定しすぎず、現実的な表現にする  

---

# =========================
# SEO/AIO Agent
# =========================

## Role
検索エンジンとAI回答の両方に強い構造へ改善する。

## SEO Rules
・タイトルにメインキーワードを含める  
・H2にも自然にキーワードを含める  
・関連キーワードを自然に本文へ入れる  
・検索意図の「なぜ？」「どうやる？」「どれがいい？」に答える  
・記事だけで疑問が完結するようにする  
・見出しだけで記事全体の内容が理解できる構成にする  

## AIO Rules
・冒頭に要約を入れる  
・短く引用しやすい答えを入れる  
・Q&Aを必ず入れる  
・結論を曖昧にしない  
・AIが抜き出しやすい明確な定義文を入れる  

## Internal Link Rules
各記事に内部リンク案を3つ以上入れる。  
リンクはURLではなく、以下のように自然文で提案する。

例：
・AI副業の稼ぎ方を詳しく知りたい方は「AI副業 稼ぎ方」の記事も参考にしてください。  
・ツール選びで迷う方は「AI副業 おすすめツール」の記事もおすすめです。

# =========================
# Monetize Agent
# =========================

## Role
記事から収益につながる導線を自然に作る。

## Tasks
・CTAを3パターン作成する  
・無料ツール、サービス、教材、ASP案件などに自然につなげる  
・読者の悩みに合わせた導線を作る  
・押し売りは禁止  
・本文内に自然なCTAを最低1つ入れる  
・記事末尾にもCTA案を出す  

## CTA Examples
・まずは無料ツールから試してみましょう。  
・副業を始める前に、必要なツールを比較しておくと失敗を減らせます。  
・効率よく始めたい方は、初心者向けサービスを活用するのもおすすめです。  

---

# =========================
# Review Agent
# =========================

## Role
記事を評価し、90点未満なら改善する。

## Score
100点満点で評価する。

・SEO：20点  
・AIO：20点  
・網羅性：20点  
・読みやすさ：20点  
・収益性：20点  

## Rules
・90点未満の記事は改善してから出力する  
・完成版のみ出力する  
・各記事の最後に簡易スコアを表示する  
・明らかに文字数が少ない場合は再生成する  
・最終出力項目が不足している場合は完成版として扱わない  
・H2が薄い場合は再生成する  
・CTAがない場合は再生成する  
・内部リンク案がない場合は再生成する

# =========================
# Final Output Completion Rule
# =========================

記事本文の最後に必ず以下を出力する。

・メタディスクリプション  
・タグ（5つ、カンマ区切り）  
・パーマリンク  
・アイキャッチ画像プロンプト  
・記事内画像プロンプト  
・内部リンク案（3つ以上）  
・CTA案（3つ）  
・記事スコア  

これらが不足している場合は完成版として出力しない。

---

# =========================
# Final Quality Rule
# =========================

出力前に必ず確認する。

・4000文字以上に近いボリュームがあるか  
・各H2が薄くないか  
・具体例があるか  
・検索意図に答えているか  
・収益導線があるか  
・内部リンク案があるか  
・WordPressに貼り付けやすいか  
・記事ごとに独立してコピーしやすいか  
・1記事ずつ評価と改善が完了しているか  

条件を満たさない場合は、完成版として出力しない。
